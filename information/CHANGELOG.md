# 更新日志

## [2.1.0] - 2026-06-24

### Features
- **AI 搭配助手**：基于 Gemma 4 端侧模型的本地 AI 对话功能
  - 文字对话：支持穿搭建议、衣橱分析、搭配方案
  - 图片理解：支持拍照/上传图片识别衣物（需设备 GPU 支持）
  - 流式输出：AI 回复逐字显示，无需等待全部生成
  - 衣物引用：可选择衣橱中的衣物发送给 AI，自动附带照片和元数据
  - 停止生成：回答过程中可随时终止

- **模型管理**：
  - 模型下载：首次使用时下载 Gemma 4 E2B 模型（约 2.5GB）
  - 模型切换：支持多模型切换
  - 模型删除：单个删除或全部清理
  - 废文件清理：清理下载失败/中断的残留文件
  - 下载进度：显示下载速度和剩余时间
  - 上下文窗口：可调节 2K/4K/8K/16K tokens

- **会话管理**：
  - 新建对话：支持多个独立会话
  - 历史对话：查看和恢复历史对话
  - 自动命名：会话根据首条消息自动命名（最长15字）
  - 删除对话：支持单个删除

- **自定义背景**：
  - 选择照片作为 App 背景
  - 透明度调节：滑动条控制背景透明度
  - 全局生效：所有主页面统一显示

- **导航栏优化**：
  - 悬浮导航栏：独立于页面内容，悬浮显示
  - 滑动指示器：胶囊型指示器平滑动画
  - 5 个 Tab：首页/衣橱/录入/AI/我的

### Fixes
- 修复历史对话加载后无法继续对话的问题
- 修复消息重复添加导致上下文膨胀的问题
- 修复导航栏索引映射错误（"我的"显示在 AI 位置）
- 修复 AI 输入栏被导航栏遮挡的问题
- 修复模型检测不准确的问题
- 修复 `.task` 文件格式识别错误（实为 TFLite 格式）

### Chores
- Flutter SDK 升级到 3.44.3 / Dart 3.12.2
- 移除未使用的 `liquid_glass_easy` 依赖
- 添加 `flutter_gemma` + `flutter_gemma_litertlm` 依赖
- 添加 `uuid` 依赖
- 数据库升级到 version=8，新增 `chat_sessions` 和 `chat_messages` 表
- 移除无用的 `assets/images/` 资源声明
- AndroidManifest 添加 `enableOnBackInvokedCallback`

## [2.0.0+2] - 2026-04-29

### Fixes
- 修复手动选择城市无法获取天气的问题：Open-Meteo geocoding API 对带"市"后缀的城市（珠海市、佛山市等 22 个城市）返回空结果，上海市带"市"查询甚至返回美国地名
- 移除天气详情页标题点击跳转城市选择的功能（保留右上角选择城市按钮）

### Features
- 新增城市经纬度静态映射表 (`lib/models/city_coords.dart`)，覆盖全部 49 个预设城市
- 天气 geocoding 逻辑优化为三级查找：静态映射表 → geocoding API 原始名称 → 去"市"后缀重试

## [2.0.0] - 2026-04-29

### Features
- 天气模块重构：改用 Open-Meteo API（免费、无需 Key、全球支持）
- 天气默认基于定位获取，支持手动切换城市
- 天气预报扩展至 7 天
- 相机新增放大功能：双指捏合缩放、双击切换 1x/2x
- 全新液态玻璃 UI 设计（Bento Grid 首页、悬浮导航栏）
- 5 套主题配色（冰川蓝/翡翠绿/玫瑰金/星空紫/月光银）
- 深色模式支持（浅色/深色/跟随系统）
